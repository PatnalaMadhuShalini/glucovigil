GlucoSmart Health Analytics

Database Schema Tables

The following tables represent the database schema for the GlucoSmart Health Analytics platform. Each table is designed to store a specific aspect of the system's data, with relationships between tables defined through primary and foreign keys.

USERS Table

Column Name Data Type Constraints Description
id SERIAL PRIMARY KEY Unique identifier for user
username VARCHAR(50) UNIQUE, NOT NULL User's chosen username
email VARCHAR(100) UNIQUE, NOT NULL User's email address
password VARCHAR(255) NOT NULL Hashed password
full_name VARCHAR(100) User's full name
phone VARCHAR(20) Contact phone number
gender VARCHAR(20) User's gender
place VARCHAR(100) User's location
achievements JSONB Array of user achievements
preferred_language VARCHAR(20) User's language preference
health_goals JSONB Array of user health goals
verification_token VARCHAR(255) Email verification token
verified BOOLEAN DEFAULT FALSE Account verification status
created_at TIMESTAMP DEFAULT NOW() Account creation timestamp
updated_at TIMESTAMP DEFAULT NOW() Last update timestamp

HEALTH_DATA Table

Column Name Data Type Constraints Description
id SERIAL PRIMARY KEY Unique identifier for health data entry
user_id INTEGER FOREIGN KEY REFERENCES users(id) Reference to user who owns this data
demographics JSONB Demographic information (age, gender, ethnicity)
physiological JSONB Physical measurements (height, weight, blood sugar, etc.)
lifestyle JSONB Lifestyle factors (exercise, diet, smoking, alcohol)
mental_health JSONB Mental health indicators (stress, sleep, anxiety)
prediction JSONB Risk assessment results and recommendations
created_at TIMESTAMP DEFAULT NOW() Timestamp of data collection
nutrition_plan JSONB Personalized nutrition recommendations
exercise_plan JSONB Personalized exercise recommendations
achievements JSONB Health achievements related to this entry
medical_records JSONB Processed data from medical documents

FEEDBACK Table

Column Name Data Type Constraints Description
id SERIAL PRIMARY KEY Unique identifier for feedback entry
user_id INTEGER FOREIGN KEY REFERENCES users(id) Reference to user who provided feedback
rating INTEGER CHECK (rating BETWEEN 1 AND 5) Numerical rating (1-5)
category VARCHAR(50) Category of feedback (UI, recommendations, etc.)
comment TEXT Detailed feedback comments
created_at TIMESTAMP DEFAULT NOW() Timestamp of feedback submission

MEDICAL_DOCUMENTS Table

Column Name Data Type Constraints Description
id SERIAL PRIMARY KEY Unique identifier for document
user_id INTEGER FOREIGN KEY REFERENCES users(id) Reference to document owner
health_data_id INTEGER FOREIGN KEY REFERENCES health_data(id) Reference to related health data entry
document_name VARCHAR(255) NOT NULL Original filename
document_type VARCHAR(50) NOT NULL MIME type or document category
document_content BYTEA Binary content of document or reference to storage
extracted_data JSONB Health data extracted from document
uploaded_at TIMESTAMP DEFAULT NOW() Upload timestamp

RECOMMENDATIONS Table

Column Name Data Type Constraints Description
id SERIAL PRIMARY KEY Unique identifier for recommendation
health_data_id INTEGER FOREIGN KEY REFERENCES health_data(id) Reference to health data that generated this recommendation
category VARCHAR(50) NOT NULL Category (diet, exercise, lifestyle, etc.)
title VARCHAR(255) NOT NULL Short title of recommendation
description TEXT NOT NULL Detailed recommendation text
priority INTEGER DEFAULT 5 Priority level (1-10, higher is more important)
implemented BOOLEAN DEFAULT FALSE Whether user has marked as implemented
created_at TIMESTAMP DEFAULT NOW() Timestamp of recommendation creation

HEALTH_REPORTS Table

Column Name Data Type Constraints Description
id SERIAL PRIMARY KEY Unique identifier for report
user_id INTEGER FOREIGN KEY REFERENCES users(id) Reference to report owner
report_type VARCHAR(50) NOT NULL Type of report (summary, comprehensive, progress)
summary TEXT Executive summary text
included_metrics JSONB Health metrics included in report
shared_with_provider BOOLEAN DEFAULT FALSE Whether shared with healthcare provider
generated_at TIMESTAMP DEFAULT NOW() Report generation timestamp

Health Data Table

Column Name Data Type Constraints Description
id UUID PRIMARY KEY Unique identifier for health record
user_id UUID FOREIGN KEY (users.id), NOT NULL Reference to user
demographics JSONB Demographic information
physiological JSONB Physical measurements
lifestyle JSONB Lifestyle factors
mental_health JSONB Mental health indicators
family_history JSONB Family health background

Feedback Table

Column Name Data Type Constraints Description
id UUID PRIMARY KEY Unique identifier for feedback
user_id UUID FOREIGN KEY (users.id), NOT NULL Reference to user
rating INTEGER NOT NULL Numerical rating (1-5)
category VARCHAR(50) NOT NULL Feedback category
comment TEXT Detailed feedback text
created_at TIMESTAMP DEFAULT NOW() Feedback submission timestamp

Documents Table

Column Name Data Type Constraints Description
id UUID PRIMARY KEY Unique identifier for document
user_id UUID FOREIGN KEY (users.id), NOT NULL Reference to user
file_name VARCHAR(255) NOT NULL Original document file name
file_type VARCHAR(50) NOT NULL Document MIME type
file_size INTEGER NOT NULL Document size in bytes
file_path VARCHAR(255) NOT NULL Storage path reference
extracted_data JSONB Data extracted from document
processing_status VARCHAR(20) DEFAULT 'pending' Document processing status
created_at TIMESTAMP DEFAULT NOW() Upload timestamp
updated_at TIMESTAMP DEFAULT NOW() Last update timestamp

Reports Table

Column Name Data Type Constraints Description
id UUID PRIMARY KEY Unique identifier for report
user_id UUID FOREIGN KEY (users.id), NOT NULL Reference to user
report_type VARCHAR(50) NOT NULL Type of report generated
file_path VARCHAR(255) NOT NULL Storage path reference
health_data_snapshot JSONB Health data used in report
created_at TIMESTAMP DEFAULT NOW() Report generation timestamp

User Sessions Table

Column Name Data Type Constraints Description
id UUID PRIMARY KEY Unique identifier for session
user_id UUID FOREIGN KEY (users.id), NOT NULL Reference to user
token VARCHAR(255) NOT NULL, UNIQUE Session token
ip_address VARCHAR(45) NOT NULL Client IP address
user_agent TEXT Client browser information
expires_at TIMESTAMP NOT NULL Session expiration time
created_at TIMESTAMP DEFAULT NOW() Session creation timestamp
last_activity TIMESTAMP DEFAULT NOW() Last activity timestamp

Health Metrics History Table

Column Name Data Type Constraints Description
id UUID PRIMARY KEY Unique identifier for metric record
user_id UUID FOREIGN KEY (users.id), NOT NULL Reference to user
metric_name VARCHAR(50) NOT NULL Name of health metric
metric_value DOUBLE PRECISION NOT NULL Value of health metric
unit VARCHAR(20) NOT NULL Unit of measurement
source VARCHAR(50) NOT NULL Source of measurement (manual, device, lab)
notes TEXT Additional context
measured_at TIMESTAMP NOT NULL Time of measurement
created_at TIMESTAMP DEFAULT NOW() Record creation timestamp
